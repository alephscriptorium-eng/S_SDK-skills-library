# Fixture sintetica

Ejemplo inventado para ejercitar el contrato del skill sin usar datos de
ningun mundo real.

## Contenido de la fixture

- `intake.md` — intake minimo con partes definidas y partes pendientes
- `resultado-esperado.md` — forma de salida esperada al materializar el
  skill

## Uso

1. Leer `SKILL.md`.
2. Leer `reference/CONTRATO.md`.
3. Aplicar el metodo sobre `intake.md`.
4. Verificar que lo no definido siga como `<pendiente>`.
